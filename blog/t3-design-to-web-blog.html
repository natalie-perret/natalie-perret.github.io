Design to Web

A responsive website automatically adjusts its content depending on the size of the screen of the device on which the user is viewing the content. It is important because there are so many different devices and screen sizes, and viewing content made for a desktop computer is very frustrating when viewing on a smartphone screen.

The mobile-first design approach stems from the view that mobile design (or design for a smartphone screen) is the hardest, and as such should be done first.

Frameworks provide a faster and easier way to code. Tasks that would usually take developers hours (or in my case weeks) and hundreds of lines of code to write, can be done in minutes with the pre-built functions that come from using a framework. Other than being faster, another benefit is the support you can get from others using the same framework as you. There are however limitations to how much you can change within the framework so you need to make sure you choose a framework that meets your needs.
